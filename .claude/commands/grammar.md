# 文法データ更新スキル

問題と回答が書かれた画像から問題を抽出して追加するスキル。

## 手順

1. まず現在のプロジェクトにある `data.js` を読んで形式・各セクションの末尾IDを把握する。

2. `$ARGUMENTS` に画像パスがあれば Read ツールで読む。なければ「画像のパスを教えてください」と聞く。

3. 画像から全問を抽出する。問題タイプ（FRAME / Exercise A / Exercise B / Exercise C）は以下の**判別ルール**で必ず判定する：

   **判別ルールは「画像内の "EXERCISE A >>" "EXERCISE B >>" "EXERCISE C >>" の見出しが最優先」。**
   見出しの下にある問題は、別の見出しが現れるまで全てそのセクション。番号が `08`〜`20` のように途中から始まっていても、見出しが「EXERCISE A」のままならExA。番号だけで勝手にExBに変えない。

   見出しが見えない・複数ページにまたがる場合は形式で判別：

   | セクション | 見た目の特徴 |
   |---|---|
   | **FRAME** | 左側に `FRAME 010` のようなラベル。番号付きで丸数字答え（①②③④） |
   | **Exercise A** | `01 ②` `02 ④` の番号 + 丸数字答え。本文は完全な英文ではなく「日本語訳」のみ表示 |
   | **Exercise B** | `01 ② hold → held` のように **「誤った語 → 正しい語」の置換形式** が必ずある |
   | **Exercise C** | 答えが**完全な英文1文**で、丸数字（①②③④）がない |

   **チェックリスト（誤判定防止）**：
   - 「→」記号があるか？ → あれば **必ずExB**
   - 完全な英文だけが書かれているか？ → **必ずExC**
   - 丸数字の答え＋日本語訳のみか？ → **ExA**（番号がいくつでも）
   - 数えた合計問題数を STEP × セクション ごとに事前に書き出してから data.js に追加する

4. 手順1で確認した形式・IDの続きで該当セクションの末尾に追加する。

5. `git add data.js && git commit -m "grammar: add questions"`

6. 何問追加したか報告する。
